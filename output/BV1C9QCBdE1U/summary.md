## 创建自定义节点

[00:00:06] 首先，我们将创建一个自定义节点，以便能够像其他节点一样通过添加节点窗口进行添加。这个自定义节点将包含生成伤害数字标签的所有逻辑。

[00:00:10] 在你的项目中，找到脚本标签。点击脚本标签后，选择“文件”菜单，然后选择“新建脚本”。在弹出的对话框中，将脚本命名为 `damage_number_spawner`，然后点击“创建”按钮。

```gdscript
extends Node2D
```

[00:00:12] 在新创建的脚本中，确保它扩展自 `Node2D`。选择 `Node2D` 作为基础类的原因是，它提供了2D空间中的位置、旋转和缩放功能，这对于我们在游戏中放置和移动伤害数字至关重要。使用 `Node2D`，你可以轻松地控制伤害数字的显示位置，并且可以利用其内置的变换功能来实现动画效果。

通过扩展 `Node2D`，你将能够在游戏场景中自由地添加和管理伤害数字，确保它们在正确的位置显示，并且能够随着游戏的进展进行动态更新。接下来，我们将在这个自定义节点中实现伤害数字生成的具体逻辑。

<a id="s3"></a>
## 设置伤害数字生成逻辑

[00:00:18] 在自定义节点中，我们将创建一个名为 `spawn_label` 的函数，用于生成伤害数字。这个函数将负责创建和配置伤害数字的标签，使其在游戏中正确显示。

[00:00:20] 首先，定义 `spawn_label` 函数，并为其添加一个参数 `critical_hit: bool = false`。这个参数的默认值为 `false`，用于指示该伤害数字是否为暴击伤害。暴击伤害通常会以不同的颜色显示，以便玩家能够一眼识别。

```gdscript
func spawn_label(damage: int, critical_hit: bool = false):
```

[00:00:30] 接下来，在函数内部，我们需要将伤害数字转换为字符串，以便在标签中显示。使用 `str()` 函数可以轻松地将整数转换为字符串格式。

```gdscript
var damage_text = str(damage)
```

[00:00:40] 现在，我们需要创建一个标签节点来显示这个伤害数字。使用 `Label` 类来创建标签，并将其文本设置为刚刚转换的 `damage_text`。

```gdscript
var label = Label.new()
label.text = damage_text
```

[00:00:50] 为了确保伤害数字标签在其他游戏元素之上显示，我们需要设置标签的 `ZIndex`。这可以通过设置标签的 `z_index` 属性来实现。将 `z_index` 设置为一个较高的值，例如 10，以确保它在其他节点之上。

```gdscript
label.z_index = 10
```

[00:01:00] 如果伤害数字是暴击伤害，我们还需要设置标签的颜色。可以使用 `Color` 类来定义颜色，并将其应用到标签的 `modulate` 属性上。以下代码将暴击伤害的颜色设置为红色（#FF0000）。

```gdscript
if critical_hit:
    var critical_hit_color = Color(1, 0, 0)  # 红色
    label.modulate = critical_hit_color
```

[00:01:10] 最后，将标签添加到当前节点中，以便它能够在游戏场景中显示。

```gdscript
add_child(label)
```

通过以上步骤，我们已经成功设置了伤害数字生成的逻辑。这个 `spawn_label` 函数将负责创建伤害数字标签，处理暴击标识，并确保标签在视觉上突出显示。接下来，我们将处理暴击和颜色变化，以进一步增强伤害数字的表现效果。
